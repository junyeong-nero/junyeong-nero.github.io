# Resume Word Counter Plan

## Goal

자소서 첨삭에 바로 사용할 수 있는 웹 기반 글자수 세기 페이지를 만든다.
단순 카운터가 아니라, 글자수 제한 관리와 기본적인 문장 점검까지 지원하는 실용 도구를 목표로 한다.

## Target Users

- 자소서 문항별 글자수를 맞춰야 하는 취업 준비생
- 첨삭 전에 문장 길이와 반복 표현을 빠르게 점검하려는 사용자
- 별도 로그인 없이 가볍게 접속해 바로 쓰고 싶은 사용자

## V1 Scope

### Core Features

- 텍스트 입력
- 실시간 글자수 계산
- 공백 포함 / 공백 제외 / 바이트 수 동시 표시
- 최대 글자수 설정
- 현재 글자수 대비 진행률 표시
- 제한 초과 시 시각적 경고
- `localStorage` 기반 자동 저장

### Editing Support

- 문장 수 계산
- 평균 문장 길이 표시
- 자주 반복된 단어 상위 목록 표시
- 공백 정리 버튼
- 전체 초기화 버튼

### Layout

- 상단: 페이지 제목과 간단한 설명
- 좌측 또는 상단: 입력 영역
- 우측 또는 하단: 통계 패널
- 하단: 반복 표현 / 문장 분석 결과

## V2 Candidates

- 문항 여러 개를 탭 또는 카드 형태로 관리
- 문항별 최대 글자수 개별 설정
- 금지 표현 / 추상 표현 하이라이트
- 수정 전후 버전 비교
- 내보내기 기능

## Technical Approach

- 정적 HTML/CSS/JavaScript로 구현
- 별도 빌드 도구 없이 GitHub Pages에서 바로 동작
- 데이터 저장은 브라우저 `localStorage` 사용
- 텍스트 분석은 클라이언트 사이드에서 처리

## File Structure Draft

```text
resume_word_counter/
  index.html
  style.css
  script.js
  plan.md
```

## Counting Rules

- 글자수:
  - 공백 포함: 입력 문자열 길이 기준
  - 공백 제외: 공백, 줄바꿈, 탭 제거 후 길이 기준
- 바이트 수:
  - UTF-8 기준으로 계산
- 문장 수:
  - `.`, `!`, `?`, `.`에 대응하는 한국어 마침표 패턴 포함 여부를 검토

## UX Principles

- 입력 즉시 결과가 바뀌어야 한다
- 초과 여부를 숫자와 색상 둘 다로 보여줘야 한다
- 모바일에서도 바로 쓸 수 있어야 한다
- 첨삭용 도구이므로 결과 패널은 읽기 쉬워야 한다

## Implementation Steps

1. `resume_word_counter/index.html` 기본 구조 작성
2. `style.css`로 반응형 레이아웃과 상태 색상 정의
3. `script.js`에서 실시간 카운팅 로직 구현
4. 자동 저장 / 복원 기능 추가
5. 반복 단어 및 문장 길이 분석 추가
6. 수동 테스트로 주요 입력 케이스 확인

## Test Checklist

- 빈 입력에서 모든 수치가 0으로 표시되는지
- 한글 / 영어 / 숫자 / 이모지 입력에서 글자수 계산이 깨지지 않는지
- 줄바꿈이 포함되어도 정상 동작하는지
- 최대 글자수 초과 시 경고가 즉시 반영되는지
- 새로고침 후 입력 내용이 복원되는지
- 모바일 폭에서도 레이아웃이 무너지지 않는지

## Open Questions

- V1에서 단일 문항만 지원할지, 다중 문항까지 바로 넣을지
- 반복 표현 분석 시 조사/불용어를 얼마나 제외할지
- 자소서 사이트별 글자수 기준 차이를 안내 문구로 넣을지
