**MCP (Model Context Protocol)**는 LLM이 **외부 도구, 데이터, 서비스와 일관된 방식으로 연결**될 수 있도록 설계된 통신 규격이다.

> “LLM이 여러 시스템과 대화하고, 행동하고, 상태를 유지하기 위한 공통 인터페이스”

---

## 1. MCP의 핵심 목적

MCP는 단순한 API 연결이 아니라 다음을 표준화하기 위해 만들어졌다.

1. **Context 통합**: 파일, DB, 로그 등을 하나의 체계로 연결
2. **Tool 호출 통합**: 외부 함수, 서비스 등을 통일된 방식으로 사용
3. **상태 관리**: multi-turn 상태, task 상태 관리
4. **Agent 협업**: 여러 agent가 공통 프로토콜로 협력

---

## 2. MCP 구조 개념도

```
[ LLM Client ]
     |  MCP Protocol
[ MCP Server ]
     ├── Tool A (Filesystem)
     ├── Tool B (Database)
     ├── Tool C (Browser)
```

LLM + MCP의 결합은 단순히 텍스트를 생성하는 존재를 넘어 **실제로 일을 하는 시스템(Agent)**으로 진화하게 한다.
