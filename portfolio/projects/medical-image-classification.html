Medical Image Classification with MedViT-V2

2025.01

üìã Project Overview

Development of a flexible fine-tuning pipeline for MedViT-v2, a state-of-the-art Medical Vision Transformer with hybrid CNN-Transformer architecture specifically optimized for medical image analysis. The pipeline supports any HuggingFace image classification dataset, enabling rapid adaptation to various medical imaging tasks including pathology, radiology, and dermatology.

üéØ Problem Definition & Goals

  • Problem: Medical image classification requires domain-specific models, but existing pipelines are often task-specific and lack flexibility for different datasets.
  • Goal 1: Create a universal fine-tuning pipeline that works with any HuggingFace medical image dataset.
  • Goal 2: Leverage MedViT-v2's hybrid architecture that combines CNN's local feature extraction with Transformer's global attention.
  • Goal 3: Provide easy configuration and training scripts for researchers with minimal deep learning experience.

  • MedViT-v2 Integration: Incorporated state-of-the-art medical vision transformer with pre-trained weights from large-scale medical imaging datasets.
  • Hybrid Architecture: CNN backbone extracts local features while Transformer layers capture global context, optimal for medical image analysis.
  • Flexible Pipeline: Dataset-agnostic design supporting any HuggingFace image classification dataset with automatic preprocessing.
  • Medical Optimization: Includes medical-specific augmentations and preprocessing suited for various imaging modalities.
  • Evaluation Suite: Comprehensive evaluation with metrics relevant to medical applications including sensitivity, specificity, and AUC-ROC.

üîß Technical Challenges & Solutions

  • Class Imbalance: Medical datasets often have severe class imbalance. Implemented weighted loss functions and oversampling strategies for rare classes.
  • Data Augmentation: Standard augmentations can be harmful for medical images. Curated medical-appropriate augmentations preserving diagnostic features.
  • Transfer Learning: Fine-tuning from general to medical domain required careful learning rate scheduling and layer freezing strategies.
  • Multi-modality Support: Different imaging modalities (X-ray, CT, dermoscopy) require different preprocessing. Implemented modality-aware preprocessing pipelines.

üìà Results & Learnings

  • Versatile Pipeline: Successfully demonstrated on multiple medical imaging datasets including skin lesion, chest X-ray, and pathology images.
  • Performance: Achieved competitive results compared to task-specific models while maintaining flexibility.
  • Reproducibility: Comprehensive documentation and configuration files enable reproducible experiments.
  • Key Learning: Gained expertise in medical imaging AI, hybrid vision architectures, and the unique challenges of healthcare ML including data handling and regulatory considerations.
