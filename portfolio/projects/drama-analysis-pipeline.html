Drama Analysis Pipeline with Multi-Agents

2025.02 - Present

üìã Project Overview

Developed during internship at CJ AI Center, this project creates a sophisticated Multi-Agentic pipeline for analyzing Korean drama scripts using LangChain. The system automates the extraction of narrative elements, character relationships, emotional arcs, and content insights from raw script data, enabling data-driven content strategy decisions.

üéØ Problem Definition & Goals

  • Problem: Analyzing drama scripts manually is time-consuming and subjective. Content teams need automated, consistent methods to extract insights from large script databases.
  • Goal 1: Build an automated pipeline that can process Korean drama scripts and extract structured information about characters, plot, and themes.
  • Goal 2: Implement multi-agent architecture where specialized agents handle different aspects of analysis (character analysis, plot summarization, emotion detection).
  • Goal 3: Create a scalable system that can process scripts in batch for content library analysis.

  • Multi-Agent Architecture: Designed specialized agents for character analysis, plot extraction, emotion tagging, and relationship mapping.

  • Structured Output: Generated JSON-formatted analysis results compatible with downstream analytics and visualization systems.

üîß Technical Challenges & Solutions

  • Script Format Variability: Korean drama scripts have inconsistent formatting. Developed robust parsers with regex patterns and heuristics to handle various formats.
  • Agent Coordination: Multiple agents needed to share context without conflicts. Implemented shared memory system and message passing protocols.
  • Context Length Limits: Long scripts exceeded LLM context windows. Created sliding window approach with intelligent chunking at scene boundaries.
  • Consistency in Analysis: Ensuring consistent character identification across script sections. Implemented entity resolution and character tracking across the entire script.
