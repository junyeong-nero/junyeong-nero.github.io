tiny-graph-RAG

2025.01 - Present

üìã Project Overview

A Knowledge Graph-based RAG (Retrieval-Augmented Generation) system that constructs structured knowledge graphs from documents and uses graph traversal algorithms for intelligent question answering. Unlike traditional vector-only RAG, this system captures entity relationships and uses BFS traversal with heuristic ranking to retrieve contextually relevant subgraphs for LLM generation.

üéØ Problem Definition & Goals

  • Problem: Traditional vector-based RAG loses relationship information between entities. When answering complex questions requiring multi-hop reasoning, vector search retrieves disconnected chunks that lack contextual coherence.
  • Goal 1: Build an automated pipeline that extracts entities and relationships from documents to construct a knowledge graph.
  • Goal 2: Implement graph-based retrieval using BFS traversal to find relevant subgraphs that preserve relationship context.
  • Goal 3: Create heuristic ranking algorithms that score retrieved context by relevance to the query.

‚öôÔ∏è Key Features & Contributions

  • Knowledge Graph Construction: LLM-based entity and relationship extraction with custom prompts optimized for structured output.
  • Graph-based Retrieval: BFS traversal starting from query-relevant entities, expanding to capture multi-hop relationships.
  • Heuristic Ranking: Custom scoring function combining entity relevance, path length, and relationship types for context prioritization.
  • Parallel Extraction: Efficient entity extraction with parallel processing for handling large document collections.
  • Dual Interface: Both CLI for batch processing and Streamlit web UI for interactive querying.

üîß Technical Challenges & Solutions

  • Entity Resolution: Same entities referenced differently across documents. Implemented fuzzy matching and embedding-based similarity for entity deduplication.
  • Relationship Quality: LLM-extracted relationships varied in quality. Added validation step and confidence scoring to filter low-quality triplets.
  • Graph Scale: Large graphs caused traversal performance issues. Implemented indexed lookups and pruning strategies for efficient BFS.
  • Context Window: Retrieved subgraphs could exceed LLM context limits. Developed summarization and compression techniques for large subgraphs.

üìà Results & Learnings

  • Improved Multi-hop QA: Graph-based retrieval significantly outperformed vector-only approach on questions requiring relationship reasoning.
  • Relationship Preservation: System maintains contextual coherence by retrieving connected subgraphs rather than disconnected chunks.
  • Visualization: Knowledge graph visualization helps users understand document structure and verify extraction quality.
  • Key Learning: Gained expertise in knowledge graph construction, graph algorithms for information retrieval, and hybrid RAG architectures.
