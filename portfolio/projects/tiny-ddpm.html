tiny-DDPM

2024.07 - 2024.09

üìã Project Overview

A from-scratch implementation of Denoising Diffusion Probabilistic Models (DDPM) designed for deep understanding of diffusion model mathematics and architecture. Beyond simple implementation, this project features production-ready code with config-based management, multiple model architectures (UNet, DiT), comprehensive testing, and support for modern sampling methods including DDIM and Classifier-Free Guidance.

üéØ Problem Definition & Goals

  • Problem: Many diffusion model implementations are either too complex to understand or too simplified to be practical. There was a need for an educational yet production-quality codebase.
  • Goal 1: Implement DDPM from scratch following the original paper to deeply understand the forward/reverse diffusion process and noise scheduling.
  • Goal 2: Extend to advanced techniques including DDIM for faster sampling and Classifier-Free Guidance for conditional generation.
  • Goal 3: Create maintainable, testable code that can serve as a foundation for further diffusion model research.

‚öôÔ∏è Key Features & Contributions

  • Multiple Model Architectures: Implemented both traditional UNet and modern Diffusion Transformer (DiT) to compare architectural approaches.
  • Config-Based Management: All hyperparameters managed via YAML config files, enabling reproducible experiments and easy parameter sweeps.
  • Advanced Sampling: DDIM implementation for 10-50x faster sampling compared to DDPM, plus CFG for controllable generation quality.
  • Apple Silicon Support: Automatic MPS detection for GPU acceleration on Apple M-series chips.
  • Comprehensive Testing: Unit tests via pytest ensuring correctness of noise scheduling, forward/reverse processes, and model outputs.

üîß Technical Challenges & Solutions

  • Variance Scheduling: Understanding the mathematical relationship between beta schedules and generation quality. Implemented linear, cosine, and learned schedules with ablation studies.
  • Timestep Embedding: Proper sinusoidal embedding implementation was crucial for model performance. Debugged by comparing with reference implementations and visualizing embeddings.
  • Memory Efficiency: Long diffusion chains caused memory issues. Implemented gradient checkpointing and optimized batch sizes for training on consumer GPUs.
  • DiT Integration: Adapting transformer architecture for image generation required careful patch embedding and positional encoding design.

üìà Results & Learnings

  • Successful Generation: Trained models successfully generate diverse, high-quality MNIST digits with proper mode coverage.
  • Sampling Speed: DDIM achieves comparable quality with 10x fewer steps, dramatically reducing generation time.
  • Educational Value: The codebase serves as a comprehensive learning resource, with detailed comments explaining the mathematics at each step.
  • Key Learning: Gained deep understanding of score-based generative models, noise prediction networks, and the connection between diffusion and score matching.
